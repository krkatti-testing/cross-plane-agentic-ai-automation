Tips for Writing Requests
Good Practices
  • Be specific about resource names
  • Mention the AWS region if needed
  • Specify environment (dev/staging/prod)
  • Include sizing requirements
  • Mention security features if needed
Supported Features
  • EKS: Node count, instance types, K8s version
  • S3: Versioning, encryption
  • RDS: Engine type, instance class, storage
  • VPC: Multi-AZ, custom CIDR
  • All: Tags, descriptions
